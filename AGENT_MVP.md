# 개인 에이전트 하네스 — MVP 스펙

## 목표
LLM API를 호출해 로컬 파일·프로젝트를 제어하는 **완전 통제형 에이전트 하네스**를 직접 만든다.
이 문서는 최소 뼈대(MVP)만 정의한다. 메시징·스킬·멀티에이전트는 이후 단계로 미룬다.

## 기술 스택
- 언어: Python 3.11+
- LLM: Anthropic API 우선 (`anthropic` SDK). 이후 다른 프로바이더로 확장할 수 있게 어댑터로 감싼다.
- 세션 저장: SQLite (표준 라이브러리 `sqlite3`)
- 도구 입출력 검증: Pydantic

## MVP 범위 (이번에 완성할 것)
"사용자 질문 → 에이전트가 파일 읽기 도구를 써서 답변"하는 **최소 ReAct 루프** 하나를 끝까지 동작시킨다.

- 도구는 딱 하나: `read_file(path)` (로컬 텍스트 파일 읽기)
- 루프: 모델 호출 → 도구 호출 요청 감지 → 도구 실행 → 결과를 대화에 이어붙임 → 최종 답변까지 반복
- 세션: 대화 turn을 SQLite에 저장하고, 재시작 후 이어서 볼 수 있어야 함

## 폴더 구조
```
agent/
├─ providers/
│  └─ anthropic_adapter.py   # 모델 호출을 통일된 인터페이스로 감쌈
├─ core/
│  └─ loop.py                # ReAct 루프 (추론 → 도구 → 관찰 반복)
├─ tools/
│  ├─ registry.py            # 도구 등록 + 스키마 노출
│  └─ read_file.py           # 첫 도구
├─ session/
│  └─ store.py               # SQLite 세션 저장/조회
├─ config.py                 # API 키, 모델명 등 설정
└─ main.py                   # CLI 진입점
```

## 설계 원칙 (지켜야 할 것)
- **프로바이더 추상화**: 루프 코드는 특정 모델 SDK를 직접 알지 못하게 하고, 어댑터를 통해서만 호출한다.
- **도구 등록 ≠ 노출**: 도구를 레지스트리에 등록하되, 루프에 넘길 때 노출 목록을 명시적으로 고른다.
- **도구 계약**: 모든 도구는 Pydantic으로 입력을 검증하고, 명확한 스키마를 반환한다.
- **얇게 시작**: 압축·멀티에이전트·메시징은 지금 구현하지 않는다. 확장 지점만 열어둔다.

## 첫 작업 순서 (이 순서로 구현)
1. `config.py` — API 키/모델 로딩
2. `providers/anthropic_adapter.py` — `call(messages, tools)` 형태의 단일 함수
3. `tools/read_file.py` + `tools/registry.py` — 도구 하나 등록
4. `core/loop.py` — 도구 호출을 처리하는 ReAct 루프
5. `session/store.py` — turn 저장/복원
6. `main.py` — CLI에서 질문을 받아 루프 실행

## 완료 기준
CLI에서 "이 파일 요약해줘: ./notes.txt" 같은 질문을 하면,
에이전트가 `read_file` 도구를 호출하고 그 내용을 바탕으로 답한 뒤, 대화가 SQLite에 남아야 한다.

## 아직 정하지 못한 것 (열어둠)
- 컨텍스트 압축 방식
- 가드레일(위험 작업 승인) 구현 시점

이 부분들은 MVP가 돈 뒤에 다시 논의한다.
